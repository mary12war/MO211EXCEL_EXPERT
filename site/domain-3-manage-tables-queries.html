Domain 3Tables and queries

Manage Tables and Queries

Turn raw data into refreshable reports with Tables, PivotTables, and imported query results.

45 minutesIntermediateTopics: PivotTables, structured references, Power Query

Real-World Hook

You have one hour to turn a messy export into a summary leaders can actually read, and every manual formula you avoid saves time.

Common Misconception

⚠️ Learners sometimes think Tables, queries, and PivotTables are separate topics, but they work best as one pipeline: clean, load, summarize, refresh.

Mastery Steps

  1. Structured tables expand more reliably and make field names easier to use in formulas and summaries.

    Task: Turn an imported list into a named Excel Table.

  2. Use Get Data to connect to another workbook or file, transform it, and load a cleaner result into Excel.

    Task: Import one external workbook and choose Load.

  3. Field areas help you compare categories fast without writing summary formulas from scratch.

    Task: Open the full PivotTables lesson to practice this workflow.

  4. Know which step needs refreshing when the source data changes so your report stays trustworthy.

    Task: Add rows to a source file and test the refresh path.

Worked Example

Import sales data from another workbook, load it into a clean worksheet, convert it to a Table if needed, and then summarize it with a PivotTable by region and category.

Use the PivotTables Mastery lesson as the fully built model for this domain's teaching pattern.

Teacher Tip

💡 Use the phrase "source, shape, summarize" so students remember the order: get the data, clean the data, then report the data.

Follow-Up Question
